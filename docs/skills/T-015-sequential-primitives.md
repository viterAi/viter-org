## Component Specs

### `SingleItemFocus`

```typescript
interface SingleItemFocusProps {
  entity: any;                      // the current item's data
  entity_type: string;              // for context (email, invoice, etc.)
  fields_to_show: FieldDisplay[];   // which fields, in what order, with what labels
  actions: ActionDefinition[];      // buttons in the action bar
  related_items?: RelatedItem[];    // optional supplementary context
  tokens?: DesignTokens;
}
```

Behavior:
- Fills most of the canvas
- Action bar at the bottom
- Optional supplementary panel (related items, links)
- Keyboard shortcuts: number keys for actions (1, 2, 3 → first, second, third action button)

### `SequenceController`

```typescript
interface SequenceControllerProps {
  queue: any[];                     // the items to process
  current_index: number;
  on_advance: (item: any, action: string) => void;
  on_skip: () => void;
  on_quit: () => void;
  child_renderer: (item: any, index: number) => ReactNode;
}
```

Behavior:
- Manages the active item index
- Renders the current item using the child_renderer
- Tracks completion (which items have been processed and how)
- Handles transitions (slide animation when advancing)
- Knows when the queue is empty → triggers completion summary

### `BriefingIntro`

```typescript
interface BriefingIntroProps {
  title: string;                    // e.g., "Email Triage"
  description: string;              // e.g., "You have 5 emails to handle"
  count: number;
  estimated_time?: string;          // e.g., "~3 minutes"
  on_start: () => void;
  on_skip_for_now: () => void;
  tokens?: DesignTokens;
}
```

### `CompletionSummary`

```typescript
interface CompletionSummaryProps {
  title: string;                    // e.g., "All caught up!"
  total_processed: number;
  breakdown: { action: string; count: number }[]; // e.g., [{ action: "Replied", count: 2 }, ...]
  on_review: () => void;            // see what was done
  on_done: () => void;
  tokens?: DesignTokens;
}
```

### `ProgressIndicator`

```typescript
interface ProgressIndicatorProps {
  current: number;                  // 1-indexed position
  total: number;
  variant: 'bar' | 'counter' | 'dots';
  show_remaining_time?: boolean;
  tokens?: DesignTokens;
}
```

## Notes for the Agent

- Use Framer Motion for transitions (already in many React stacks; if not, install it).
- Action bar: render as a horizontal row of buttons, primary action visually distinct, secondary actions standard.
- Don't build drag-and-drop reordering of the queue. That's not a sequential view feature.
- Empty state when queue is empty: don't crash, show the completion summary (or a generic "nothing to do" if the user got there without processing items).
- Handle the case where `sequence-controller` has zero items: show an empty briefing intro, not an error.
